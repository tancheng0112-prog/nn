# 基于强化学习的人形机器人平衡控制仿真

## 项目概述
本项目基于深度强化学习（SAC）算法，在 MuJoCo 物理仿真环境中实现人形机器人的动态平衡与行走控制。项目重点解决高维观测空间（376 维）下的策略训练、环境跨版本兼容与预训练权重部署等工程化问题，为后续稳定行走与复杂运动控制奠定基础。

## 项目目标
- **仿真环境搭建**：配置 MuJoCo 物理引擎，解决跨版本（v4）兼容性与渲染问题。
- **策略模型部署**：实现 SAC 预训练模型的解压、兼容加载与实时推理。
- **稳定性优化**：修复物理引擎属性冲突，平滑动作输出，使机器人从“原地扭动”过渡到稳定站立与初步行走。
核心任务：在维持身体平衡的前提下，实现稳定的步态行走

## 一、主要改进与技术突破

### 1. 代码架构模块化重构
- 将臃肿的测试脚本拆分为 **`env_mujoco.py`（环境定义与补丁）**、**`train.py`（强化学习训练）**、**`humanoid_walk.py`（演示部署）**，职责清晰、便于维护。
- 引入 **`zipfile` 动态解压机制**，直接从 `.zip` 加载 `.pth` 权重，提升模型分发与部署效率。

### 2. MuJoCo 跨版本环境适配
- **API 动态补丁**：针对新版 MuJoCo 移除 `solver_iter` 的问题，通过 `hasattr` 动态映射到 `solver_niter`，保证代码在 v3/v4 均可运行。
- **Gymnasium 接口对齐**：适配 `Humanoid-v4`，修正 `frame_skip` 等参数传递冲突，保证观测与动作空间一致性。

### 3. 权重加载与数值稳定性优化
- **非严格权重加载（strict=False）**：解决跨库版本导致的 `Missing key(s)` 错误，强制加载 Actor/Critic 核心权重。
- **动作限幅与平滑**：加入 `action_scale` 因子，抑制初期控制过冲，减少高频抖动与原地抽动。
- **推理确定性控制**：开启 `deterministic=True`，降低动作噪声，提升运动连贯性。

## 五、算法方法与后续规划
1. 当前采用方法：

   SAC (Soft Actor-Critic)：采用基于最大熵的强化学习算法，相比于 PPO，它在连续动作控制上更具探索性，且样本效率更高。

   动作平滑处理：在推理端引入了 action_scale 因子，防止关节因输出过大而导致物理仿真崩溃。

   确定性策略推理：在测试阶段关闭探索噪声（deterministic=True），确保机器人动作的连贯性。

2. 后期规划 ：
   课程学习 (Curriculum Learning)：计划先训练站立平衡，再逐步过渡到行走和跑步。

   奖励函数精修：目前的奖励主要基于前移距离，后续将加入足端压力（Foot Pressure）反馈，以减少“原地抽动”现象。


## 六、核心技术难点与解决历程

### 1. 环境属性冲突（solver_iter 报错）
- 问题：MuJoCo v4 移除 `solver_iter` 属性导致初始化失败。
- 解决：动态补丁映射 `solver_iter → solver_niter`，兼容新旧版本。

### 2. 权重加载失败（PermissionError / 路径错误）
- 问题：直接读取目录而非权重文件，或权限不足。
- 解决：使用 `zipfile` 自动解压，路径归一化，避免硬编码。

### 3. 模型参数不匹配（Missing key(s)）
- 问题：不同训练环境导致权重键名不一致。
- 解决：`torch.load(..., strict=False)`，保留核心网络权重，忽略不匹配分支。

### 4. 机器人原地高频抖动
- 问题：动作输出幅值过大、策略随机性高。
- 解决：动作限幅 + 确定性推理，显著降低震荡。

## 七、系统技术架构与核心模块说明
本项目采用典型的“环境-模型-部署”三层架构：
```
[MuJoCo 物理引擎] ↔ [Gymnasium 接口层]
        ↑                    ↓
[部署与渲染层] ↔ [SB3-SAC 决策层]
```
- MuJoCo：提供刚体动力学与碰撞仿真。
- Gymnasium：输出 376 维观测向量，接收动作指令。
- SB3-SAC：训练/加载策略网络，输出连续力矩动作。
- Deployment：自动解压、权重兼容、实时渲染。
- **`humanoid_walk.py`**：入口脚本，负责解压、加载、推理与渲染循环。
- **`env_mujoco.py`**：环境补丁、参数适配与渲染修复。
- **`train.py`**：SAC 算法训练、超参数配置与日志记录。

## 八、运行结果、不足与优化方向

### 8.1 运行结果
- 完整链路可复现，机器人能在仿真环境中站立并执行基础动作。
- 渲染流畅，物理反馈真实，可用于后续策略迭代与算法对比。

### 8.2 现存不足
- **步态稳定性有限**：受跨版本权重差异影响，机器人仍存在高频小幅度抖动，未实现长距离稳定行走。
- **部分网络权重缺失**：辅助网络（如价值网络分支）因键名不匹配未加载，影响策略最优性。

### 8.3 后续优化方向
1. **在线微调（Fine-tuning）**：基于当前权重在 `Humanoid-v4` 继续训练 100k 步，对齐参数并适应新版物理特性。
2. **课程学习（Curriculum Learning）**：先训练平衡，再逐步过渡到行走、跑步等复杂运动。
3. **奖励函数优化**：在前移奖励基础上加入足端压力、关节平滑项，抑制原地抽动。
4. **位置控制引入**：从力矩控制升级为准直接位置控制，提升关节刚度与运动精度。

## 九、总结
本项目通过**模块化重构、跨版本适配、兼容权重加载与动作平滑**，系统性解决了人形机器人强化学习部署中的关键工程问题。尽管当前步态仍有优化空间，但技术架构已完全验证，为后续大规模训练与复杂运动控制提供了可靠基础。
